# Windows setup notes

## Why this matters

A lot of local LLM documentation is Linux-first.
This repo includes notes from a Windows-based setup because that is what many users actually run.

## Common pain points

- backend installation friction
- CUDA version mismatch
- incomplete documentation
- memory-related crashes
- inconsistent performance expectations

## What helps

- keeping setups simple
- testing one backend at a time
- documenting exact model + quant + context combinations
- using repeatable prompts when comparing runs
