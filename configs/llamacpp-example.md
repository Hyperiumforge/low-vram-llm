# llama.cpp notes

## Why use it

llama.cpp is one of the most practical local inference tools because of its flexibility and broad hardware support.

## What I look at

- GGUF compatibility
- CPU/GPU balance
- memory usage
- context tradeoffs
- ease of getting a first usable result

## Practical question

For constrained hardware, the best backend is not always the fastest backend.
Sometimes the best backend is the one that is easiest to run consistently.
