# ExLlamaV2 notes

## Why use it

ExLlamaV2 is often interesting for low-VRAM users because it can offer strong performance for supported model formats.

## What I test for

- load success
- usable tokens/sec
- coding prompt quality
- memory pressure during longer prompts

## Evaluation checklist

- Does the model load reliably?
- Does generation remain stable after multiple prompts?
- Is coding quality still acceptable at the chosen quantization?
- Is the setup simpler or harder than alternatives?
